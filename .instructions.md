# Dayli Project Instructions

## Overview

Dayli is an AI-powered trading bot for Manifold Markets. This file contains instructions for working on the project in VS Code with GitHub Copilot.

## Project Context

- **Purpose**: Automated prediction market trading bot with real-time dashboard
- **References**: 
  - Dashboard UI: https://2ndpairofboots.github.io/dayli/
  - API Docs: https://github.com/Thomas-Lemoine/manifold-markets-skill
- **Tech Stack**: Python/Node.js backend, React/Vue frontend, Manifold Markets API

## Development Workflow

### File Organization
- `/docs` - Project documentation and architecture
- `/backend` - Trading bot logic and API integration
- `/frontend` - Dashboard UI components
- `/config` - Configuration files and templates
- `/tests` - Test suite

### Key Modules to Build

1. **API Client** (`backend/src/api/`)
   - REST client for market data, bets, portfolio
   - WebSocket client for real-time updates
   - Rate limiting & error handling

2. **Strategy Engine** (`backend/src/strategy/`)
   - Market analysis & prediction model
   - Trade suggestions generation
   - Configurable thresholds

3. **Trade Execution** (`backend/src/trading/`)
   - Bet placement (standard, limit, multi-outcome)
   - Position management
   - Risk controls (slip cap, max drawdown)

4. **Portfolio Manager** (`backend/src/portfolio/`)
   - Balance tracking
   - P&L calculation (1D, 1W, 1M, LIFE)
   - Win rate statistics
   - Position queries

5. **Scheduler** (`backend/src/scheduler/`)
   - Periodic bot execution
   - Event-driven triggers
   - Error recovery

6. **Dashboard UI** (`frontend/src/`)
   - Metrics panel (balance, P&L, win rate)
   - Charts (time-series, strategy breakdown)
   - Markets browser
   - Recent trades log
   - Bot suggestions panel

## Coding Standards

### Python
- PEP 8 style (use `black` for formatting)
- Type hints on all functions
- Async/await for I/O operations
- Logging via `logging` module
- Error classes for custom exceptions

### JavaScript/TypeScript
- ESLint + Prettier for formatting
- TypeScript for type safety
- React hooks best practices
- Component composition over large components
- Custom hooks for logic reuse

### API Integration
- Never hardcode API keys (use `.env`)
- Implement retry logic with exponential backoff
- Cache market data when possible
- Validate all API responses
- Log API calls for debugging

## Testing Strategy

1. **Unit Tests**: Individual component testing
2. **Integration Tests**: Module interaction testing
3. **Paper Trading Tests**: Simulate trades without real money
4. **Live Validation**: Small position sizes initially

## Common Tasks

### Adding a New Market Data Endpoint
1. Define REST method in `backend/src/api/rest_client.py`
2. Add rate limiting if needed
3. Add parsing logic for response
4. Cache if applicable
5. Add unit test

### Adding Dashboard Metric
1. Create API endpoint in backend
2. Create React component in frontend
3. Add data fetching hook
4. Style with Tailwind/CSS
5. Add error handling

### Extending Strategy Model
1. Add analysis method in `backend/src/strategy/analyzer.py`
2. Implement prediction logic
3. Add threshold configuration
4. Test with sample data
5. Log decisions for debugging

## Debugging Tips

- Enable logging: Set `LOG_LEVEL=debug` in `.env`
- Use VS Code debugger for Python/Node.js
- Check browser DevTools for frontend issues
- Review API response payloads in logs
- Verify `.env` configuration
